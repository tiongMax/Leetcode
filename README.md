# LeetCode Practice

This repository is my personal collection of LeetCode solutions, notes, and revision material.
It is organized by topic area so I can track solved problems, revisit harder ones, and build stronger pattern recognition over time.

## Repository Structure

- `DSA/` - Data structures and algorithms practice
- `SQL/` - SQL query practice

Each section may contain folders such as:

- `done/` - Problems already solved
- `to_revise/` - Problems to revisit and practice again
- `to_memorize/` - Important patterns or solutions worth remembering

## Main Focus

This repo is mainly used for:

- Practicing interview-style coding problems
- Tracking solved LeetCode questions
- Reviewing common algorithms and SQL patterns
- Keeping reusable notes and observations
- Building speed and confidence through repetition

## Languages Used

Solutions may be written in:

- Python
- C++
- Java
- TypeScript
- SQL

## Naming Convention

Most files follow this pattern:

```text
problemNumber - problemName.extension
```

Examples:

```text
200 - numberOfIsland.py
994 - rottingOrange.py
695 - maxAreaIsland.py
```

## Study Workflow

1. Solve the problem independently.
2. Save the solution in the correct folder.
3. Move difficult problems into `to_revise/`.
4. Move key patterns into `to_memorize/`.
5. Revisit older problems until the approach feels natural.

## Goal

The goal of this repository is to turn repeated practice into reliable problem-solving patterns for coding interviews and technical assessments.
